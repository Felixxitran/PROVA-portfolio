Case Study 02 — SEO & Performance Optimization

Codelynx: 40% Traffic Growth Through Technical SEO

Optimized an existing WordPress e-commerce site for search performance, Core Web Vitals, and conversion, resulting in measurable traffic and revenue impact.

The Challenge

The client's WordPress site was underperforming in search — slow load times, poor mobile scores, and product pages that weren't ranking for target keywords. They needed someone to fix the technical foundation and improve the conversion funnel.

What I Did

  • Core Web Vitals audit — identified LCP bottlenecks (unoptimized images, render-blocking CSS) and CLS issues (layout shifts from lazy-loaded elements)
  • SEO overhaul — rewrote meta titles/descriptions with target keywords, added structured data markup, fixed broken internal links, created XML sitemap
  • Conversion optimization — redesigned landing pages with clearer CTAs, ran A/B tests on page layouts, improved mobile checkout UX
  • Stripe integration & security — configured payment gateway, SSL certificate, and privacy policy pages for compliance
  • Performance monitoring — set up Google Analytics (GA4) and Search Console tracking to measure impact

Technical Details

Example: Core Web Vitals Optimization Approach
/* Before: LCP = 4.2s, CLS = 0.25 (poor) */
/* After:  LCP = 1.8s, CLS = 0.05 (good) */

/* 1. Image optimization */
<img src="product.webp"
     width="600" height="400"  <!-- explicit dimensions prevent CLS -->
     loading="lazy"
     decoding="async"
     alt="WithinStride smart insole - side view">

/* 2. Critical CSS inlined, non-critical deferred */
<link rel="preload" href="critical.css" as="style">

/* 3. Structured data for product pages */
<script type="application/ld+json">
{
  "@type": "Product",
  "name": "WithinStride Smart Insole",
  "offers": { "price": "199.00", "priceCurrency": "CAD" }
}
</script>

Results

  • 40% increase in organic traffic within 3 months
  • LCP improved from 4.2s to 1.8s (Good rating in Google PageSpeed)
  • CLS reduced from 0.25 to 0.05 (eliminated layout shift issues)
  • A/B testing identified page layouts that improved click-through by measurable margins

My approach to e-commerce SEO

Not just "I know SEO" — here is specifically how I think about optimizing a product like WithinStride.

🎯

Keyword Strategy

Research buyer-intent keywords ("smart insole for gait analysis", "foot pressure sensor insole"). Map keywords to specific pages — product page, landing page, blog.

📄

On-Page Optimization

Unique meta titles/descriptions per page. H1/H2 hierarchy. Alt text on all images. Internal linking between product and educational content.

⚡

Technical Performance

Core Web Vitals (LCP under 2.5s, CLS under 0.1). Image compression (WebP). Lazy loading. Critical CSS inlining. Mobile-first responsive design.

📊

Structured Data

Product schema markup for rich snippets in search results. FAQ schema for common questions. BreadcrumbList for site navigation.

🔍

Search Console & GA4

Track impressions, CTR, and rankings per keyword. Monitor crawl errors and indexing. Set up conversion goals and e-commerce tracking in GA4.

🧪

A/B Testing & CRO

Test headlines, CTA buttons, page layouts. Track conversion rates per variant. Iterate based on data, not guesses.
